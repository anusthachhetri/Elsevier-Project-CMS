Satellite RDF and XHTML Validation in Vtool

Vtool version: 5.93.1

Date: 24 December 2024

Introduction

This document describes the procedures to be followed to validate stand-alone RDF Satellite XML files, XHTML files, and A300 and TFS datasets with the Vtool.

Procedure

In order to validate an RDF Satellite XML file or an A300 dataset, the user needs to create a folder by name satellite-schemas in the same folder where the vtool.jar file is located. The satellite-schemas folder should contain all the schema files (these schema files may be in subfolders) that are required to validate the input XML file. This folder should contain an XML catalog file named satellite-catalog.xml. The main schema should have a valid entry in the catalog. A satellite-catalog.xml example is given below:

<?xml version="1.0" encoding="iso-8859-1"?>

<catalog xmlns="urn:oasis:names:tc:entity:xmlns:xml:catalog" prefer="public">

<group>

<system systemId="http://data.elsevier.com/schema/GenHosp-CSP-1/gh_v01.xsd" uri="GenHosp-CSP-1/gh_v01.xsd"/>

<system systemId="http://data.elsevier.com/schema/Lancet-CSP-1/lnb_v09.xsd" uri="Lancet-CSP-1/lnb_v09.xsd"/>

<system systemId="http://data.elsevier.com/schema/QAA-CSP-1/qaacsp_v01.xsd" uri="QAA-CSP-1/qaacsp_v01.xsd"/>

<system systemId="http://data.elsevier.com/schema/SKOSsatellite-1/voc_v01.xsd" uri="SKOSsatellite-1/voc_v01.xsd"/>

<systemSuffix systemIdSuffix="gh_v02.xsd" uri="gh_v02.xsd"/>

</group>

</catalog>

You will note in the above XML that there are two main elements, namely, system and systemSuffix. The system element should be used when the schemaLocation attribute of the root element of the RDF XML file has the schema path along with the default namespace URI. The systemSuffix element should be used when the schemaLocation attribute of the root element contains only the schema name without the default namespace URI. For further clarification about the catalog please visit : http://www.oasis-open.org/committees/download.php/14809/xml-catalogs.html.

In order to validate XHTML files or TFS datasets, a similar procedure should be followed. The only difference is in the folder/file naming. The folder name should be xhtml -schemas and the corresponding catalog XML file name should be xhtml-catalog.xml.

To detect the RDF errors, Vtool is now available with JENA parser (a Semantic Web Framework), use the option "-rdf" along with the regular Vtool options both while validating an individual RDF XML file or a dataset.

To validate an individual RDF XML file, use the option "-satellite" along with the regular Vtool options.

To validate individual RDF XML against both JENA parser and Vtool Satellite both options "-rdf" "-satellite" should be used.

To validate XHTML files, the option "-xhtml" should be used along with the regular Vtool options.

From Vtool v5.25.0 onwards, an additional option "-v" is introduced to facilitate parsing of very large files (tested with a thesaurus RDF Elsevier Satellite XML of size 600 MB) and other options such as "-satellite" and "-rdf" can also be used in addition to this option "-v".

When an A300 or a TFS dataset.xml file is given as input to the Vtool, additional command options are not required. The Vtool will automatically enable those options in run-time.
